# Agent IM 官网 / 入口开发规划（EN / 繁中 / 简中）

日期：2026-03-15  
面向对象：官网与入口全栈开发工程师  
阶段：Phase 1 并行线（不阻塞主线研发）

---

## 1. 目标与定位

官网/入口不是品牌展示站，而是三合一系统：

1. **战略叙事入口**：10 秒内讲清楚核心价值  
2. **转化入口**：60 秒内引导用户完成下一步  
3. **开发者入口**：快速接入文档与示例

对齐主叙事：

**可信调用的 Agent 协作网络**（不是“大而全 Agent 平台”）。

---

## 2. 受众与主动作

## 2.1 首批受众

- 开发者型小团队
- 早期企业创新团队
- 对可信协作敏感的 AI 工作流团队

## 2.2 每类受众的主动作（Primary CTA）

- 开发者：`Start Building`（进入 docs / quickstart）
- 团队负责人：`Book a Demo`
- 泛兴趣用户：`Join Waitlist`

---

## 3. 范围与非范围

## 3.1 本阶段范围（做）

- 官网首页
- 开发者入口页
- 场景/用例页（2-3 个）
- 信任与安全说明页
- Waitlist / Demo 预约页
- 基础埋点与漏斗看板
- EN / 繁中 / 简中三语内容框架

## 3.2 本阶段非范围（不做）

- 重 CMS、复杂营销自动化
- 大量动画与高维护交互特效
- 全量博客体系
- 复杂账户系统改造
- 与主线后端深耦合的新业务接口

---

## 4. 里程碑节奏（6 周）

## W1：信息架构与叙事对齐

- 定稿 IA 与页面树
- 定稿一屏定位文案（EN/TW/CN）
- 完成设计 token 与组件骨架

## W2-W3：核心页面开发

- 首页、开发者入口、信任页上线可用版本
- Waitlist / Demo 表单打通
- 三语路由与语言切换可用

## W4：转化与埋点

- 定义并接入核心事件埋点
- 完成漏斗看板第一版
- 修复关键路径障碍（移动端优先）

## W5：内容增强与案例页

- 发布 2-3 个高价值场景页
- 补充 FAQ 与对比说明（为什么可信调用）

## W6：复盘与迭代计划

- 产出数据复盘报告
- 给出下一周期实验计划（文案/CTA/路径）

---

## 5. 技术建议（默认栈）

- 框架：Next.js + TypeScript strict
- 样式：Tailwind + 统一 design tokens
- 内容：MD/MDX 或轻量 CMS（后置）
- i18n：路由级国际化（`/en` `/zh-Hant` `/zh-Hans`）
- 数据：埋点 + 简单 BI 看板（可先 PostHog / GA4 任一）
- 部署：Vercel 或等效平台（优先快速迭代）

---

## 6. 质量门槛（DoD）

每次发布必须满足：

- 三语页面关键路径可用
- 首屏可在短时间理解价值与动作
- CTA 可达、表单可提交、事件可追踪
- 移动端与桌面端关键页面无阻塞问题
- 页面承诺与产品真实能力一致

---

## 7. 协作边界（与主线工程）

- 官网线仅消费稳定、脱敏、只读接口
- 不要求主线临时开发高耦合接口
- 所有对外能力表达，必须与主线可交付版本一致
- 争议优先级：可信闭环 > 入口体验 > 页面美化
